% Abstract
\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}

This report presents a comprehensive software testing analysis of the \textbf{Online Food Delivery System} --- a web-based platform designed to connect customers with local restaurants for seamless food ordering and delivery services. The system enables users to browse restaurant menus, place orders, make secure payments, track deliveries in real-time, and provide feedback on their experience.

The primary objective of this project was to develop a robust, scalable, and user-friendly platform while ensuring high quality through systematic testing methodologies. This report documents the testing strategy, test case design, execution results, defect analysis, and tools utilized throughout the Software Testing course project.

\textbf{Keywords:} Online Food Delivery, Software Testing, Test Automation, Web Application Testing, Quality Assurance, Functional Testing, Performance Testing

\section{System Architecture}
The Online Food Delivery System follows a three-tier architecture:

+------------------+------------------+------------------+

\section{Functional Modules}

\subsection{Customer Module}
\begin{itemize}[leftmargin=*]
    \item \textbf{User Registration \& Login:} Secure account creation with email/phone verification
    \item \textbf{Profile Management:} Update personal details, addresses, and preferences
    \item \textbf{Restaurant Discovery:} Search and filter restaurants by cuisine, rating, location
    \item \textbf{Menu Browsing:} View detailed menus with images, descriptions, and pricing
    \item \textbf{Cart Management:} Add/remove items, apply promo codes, view totals
    \item \textbf{Order Placement:} Checkout process with multiple payment options
    \item \textbf{Order Tracking:} Real-time tracking of order status and delivery location
    \item \textbf{Order History:} View past orders and reorder functionality
    \item \textbf{Ratings \& Reviews:} Submit feedback for restaurants and delivery experience
\end{itemize}

\subsection{Restaurant Module}
\begin{itemize}[leftmargin=*]
    \item \textbf{Restaurant Registration:} Onboarding process for restaurant partners
    \item \textbf{Menu Management:} Add, edit, remove menu items and categories
    \item \textbf{Order Management:} Accept/reject orders, update preparation status
    \item \textbf{Analytics Dashboard:} View sales reports, popular items, customer insights
    \item \textbf{Profile Settings:} Update restaurant details, operating hours, delivery radius
\end{itemize}

\subsection{Admin Module}
\begin{itemize}[leftmargin=*]
    \item \textbf{User Management:} Manage customer and restaurant accounts
    \item \textbf{Content Moderation:} Review and approve restaurant listings
    \item \textbf{Order Oversight:} Monitor all platform orders and resolve disputes
    \item \textbf{System Configuration:} Manage payment gateways, delivery charges, promotions
    \item \textbf{Reporting:} Generate comprehensive platform analytics
\end{itemize}

\subsection{Delivery Module}
\begin{itemize}[leftmargin=*]
    \item \textbf{Delivery Partner Registration:} Driver onboarding and verification
    \item \textbf{Order Assignment:} Automated assignment based on proximity
    \item \textbf{Route Optimization:} Navigation assistance for efficient delivery
    \item \textbf{Earnings Tracking:} View completed deliveries and earnings
\end{itemize}

\section{Test Objectives}
\begin{enumerate}
    \item Validate all functional requirements specified in the SRS document
    \item Ensure system meets non-functional requirements (performance, security, usability)
    \item Identify and document defects for resolution
    \item Verify integration with external systems
    \item Confirm system readiness for production deployment
\end{enumerate}

\section{Test Scope}

\textbf{In Scope:}
\begin{itemize}
    \item All user-facing features (Customer, Restaurant, Admin, Delivery)
    \item Backend APIs and business logic
    \item Database operations and data integrity
    \item Third-party integrations (Payment, SMS, Maps)
    \item Security mechanisms and access controls
    \item Performance under expected load
\end{itemize}

\textbf{Out of Scope:}
\begin{itemize}
    \item Load testing beyond 3x expected capacity
    \item Penetration testing by external security firms
    \item Testing on legacy browsers (IE11 and below)
